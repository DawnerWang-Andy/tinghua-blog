## 当前实际可用的工具和技能

### 1. `graphify` — 知识图谱

把小说文件转成交互式知识图谱，适合检查人物关系密度、意象分布、伏笔回收。

```bash
/graphify novel/听话 --directed --wiki
```

可以发现：
- 哪些人物关系最密集（目前几乎全部集中在陈远平-小野）
- 门、椅子、笔记本、鱼肚、书、光等意象在各章的分布
- 哪些伏笔在图谱上没有回收边

### 2. `nature-polishing` — 学术润色（可借用做文学语言打磨）

虽是为学术论文设计的，但其"减少冗余、增加精确度"的逻辑同样适用于小说语言打磨。适合用来：
- 检查抽象词密度（"控制""自主""系统""合理"等）
- 把部分解释性句子替换为动作描写

### 3. `literature-note` — 结构化提取

可以反过来用：把每章的核心意象、不可逆变化、物件动作提取成结构化卡片，方便横向对比各章功能是否重叠。

### 4. 通用 Agent 分工

| 任务 | 工具 | 用法 |
|------|------|------|
| 逐章功能诊断 | Claude Code 直接阅读 | 读一章，输出：本章不可逆变化是什么、意象推进了什么 |
| 人物关系密度检查 | `graphify` | 生成图谱，看是否有角色只出现一次就消失 |
| 语言层面打磨 | `nature-polishing` 逻辑借用 | 标注高频抽象词，建议替换为动作 |
| 结构修订计划 | 手动维护 `04-修改建议.md` 的优先级表 | 每改完一项就打勾 |

## 推荐修订顺序

### 第一轮：结构（最重要）

目标：
- 压缩第 2-7 章功能重叠的部分（保留代表性场景，合并相似证明）
- 确认每章有一个不可逆变化
- 给第 21、23、25 章增加现场阻力和误读

### 第二轮：人物

目标：
- 给何哥、苏同学增加回收（让他们在后文再出现一次）
- 控制陈远平后期自省的语言尺度（多动作，少理论）
- 增强周秀兰和陈建国的差异（目前陈建国偏符号化）

### 第三轮：意象回收

目标：
- 琥珀：在第 30 章回收（写作既是保存也是封存）
- 鱼肚：在第 29 或 30 章轻回收（不再分好坏位置）
- 苏同学：后文让她读到《听话》，给小野一个反馈

### 第四轮：语言打磨

目标：
- 减少"控制""自主""系统""合理"等抽象词的出现频率
- 用动作替代解释（"手指停在弹簧扣上"替代"他在控制"）
- 统一叙事时态和视角切换的标记方式

## 修订追踪

建议在 `04-修改建议.md` 的每一条后面加 `[ ]` 复选框，改完打勾：

```markdown
- [x] 1. 压缩第 2-7 章重复机制
- [ ] 2. 给第 21-25 章增加现场阻力
- [ ] 3. 增强小野外部关系
- [ ] 4. 减少陈远平直接理论化自省
- [ ] 5. 回收琥珀、鱼肚、苏同学等已有伏笔
```

## 注意事项

- 先改结构，再改句子。结构不对，改句子白费。
- 每次只改一章，改完重读一遍确认没有破坏原有节奏。
- 陈远平越真诚越可怕——不要把他改成"坏人"。
- 结尾不需要大和解。两杯水都喝掉已经够了。
