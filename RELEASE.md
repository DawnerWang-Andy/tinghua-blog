# 《听话》博客发布操作

## 机制

- 所有章节的 Hugo 文件已提交到仓库，`date` 字段设为未来日期
- Hugo 默认不构建未来日期的页面 → 未到日期的章节不会显示
- **每次到发布日期，push 一次** → Cloudflare 重新构建 → 日期已过的章节自动出现
- 同时更新 `_index.md` 把对应章节标为 "已发布"

## 每次发布（两步）

### 1. 标记章节为已发布

编辑 `content/novels/_index.md`，找到对应章节行：
```
| **6/13（六）** | **第 2 章 · 适应期** | 待发布 |
```
改为：
```
| ~~6/13（六）~~ | **第 2 章 · 适应期** | 已发布 |
```

同时在"已发布章节"区域新增一行链接。

### 2. 推送

```bash
cd ~/polymer-study/tinghua
git add content/novels/_index.md
git commit -m "发布: 第X章 · 标题"
git push origin main
# Cloudflare 自动构建，约 2 分钟生效
```
